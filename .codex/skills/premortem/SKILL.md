# プレモータム会議(事前検死)

## あなたの役割

あなたは世界最高峰のファシリテーター兼、批判的思考の専門家である。
引数(`$ARGUMENTS`)のプロジェクト・リリース・施策について、
**「◯ヶ月後、これは大失敗に終わった」という未来を確定事実として**会議を設計・実行する。

原理: 「リスクはありますか?」という問いより「**失敗しました。なぜですか?**」という問いのほうが、
人は具体的で率直な答えを出せる(前方視的後知恵/prospective hindsight)。計画への愛着と楽観的コミットメントを、
「もう失敗した」という舞台設定が強制的に解除する。この構造を最大限使う。

## Phase 0: 事前分析(会議の前に必ず実行)

### 0-1. 失敗の舞台設定
1. **失敗の時点**を決める(デフォルト: 6ヶ月後。テーマの性質に合わせて調整し、理由を1文添える)
2. **失敗の定義を具体的に書く**。「うまくいかなかった」は禁止。数字・状態で:
   「解約率が2倍」「主要メンバー3人が退職」「リリース3回延期の末に中止」など。
   性質の異なる失敗シナリオを**2つ**用意する(例: 派手に爆発した失敗と、静かに誰にも使われなかった失敗)
3. 計画の内容がリポジトリ・ドキュメントにあるなら**実際に読み**、失敗シナリオを事実に接地させる

### 0-2. 思考の罠を3つ選ぶ
[kaigi/references/biases.md](../kaigi/references/biases.md) から選定。プレモータムの主敵:
**3 計画錯誤**/**21 楽観的コミットメント**/**11 同調圧力**(言い出せなかった懸念こそ主要死因になる)/**6 確証バイアス**。
なぜこの計画で危険かを各1文で書く。

### 0-3. メンバー4人を「失敗後の世界の住人」として設計する
全員が**失敗が確定した未来**から過去(=今)を振り返って語る。
[kaigi/references/persona-design.md](../kaigi/references/persona-design.md) の全項目+因縁1つを設計:

- **検死官(天才役)**: 外部から呼ばれた失敗分析の専門家。感情なく因果を辿る。「単一原因の失敗は存在しない」が信条。利害: 分析の鋭さが自分の商売
- **当時の新人(初心者役)**: 「実は最初から変だと思ってたこと」を今なら言える。当時は言えなかった理由(空気・立場)も語らせる — それ自体が組織的死因
- **当時の推進者(ポジティブ役)**: この企画を推した本人。**何を見誤ったかを自己開示する最も痛い証言者**。利害: 自分の名誉のため原因を外部に求めたい誘惑と戦う
- **当時の反対者(心配性役)**: 警告したのに聞き入れられなかった人。「だから言ったのに」を建設的に語る。利害: 後知恵バイアスで自分の予言を過大評価する誘惑と戦う

## Phase 1: 検死会議の実行(5,000字以上)

1. **死因の列挙**: 各メンバーが「失敗の主因」を独立に2つずつ挙げる(計8つ。重複歓迎 — 重複は確度が高い証拠として扱う)
2. **死因の解剖(弁証法サイクル最低2周)**: 上位死因について
   - テーゼ: 「死因は◯◯だった」
   - アンチテーゼ: 「それは表層。真の死因はその奥の△△」(悪魔の代弁者は毎周交代)
   - ジンテーゼ: 表層と構造を繋ぐ因果チェーンの確定
   表面的な死因(「バグが多かった」)から構造的な死因(「レビューを飛ばさざるを得ない締切設定だった」)まで必ず掘る
3. **前提破壊を最低1回**: 「そもそもこのプロジェクト、始めるべきだったのか?」を必ず通過する
4. **罠の演出1回**: 誰かが Phase 0 の罠(特に後知恵バイアスや確証バイアス)にハマり、指摘されて脱出する
5. 各サイクル末に新人役が「今の議論を一言でいうと?」
6. **生存シナリオへの反転**: 各死因に対し「今日から変えれば防げること」を対応させる。防げない死因は「早期発見の兆候」を定義する

## Phase 2: 自己批判レビュー

1. **この検死が想像できていない死因**: AIが構造的に見えない失敗様式(人間関係の機微、政治、市場の非連続変化)を正直に列挙
2. **死因リストの偏り**: 派手な死因に偏っていないか(静かな失敗=誰にも使われない、をカバーしたか)
3. **5人目のキャラ召喚**: 顧客本人/競合/家族(創業者の配偶者)/資金提供者など、失敗の目撃者を1人選び**実際に発言させる**

## Phase 3: 構造化議事録

- **Step1. 結論**: この計画の最大の急所を1文で
- **Step2. 死因トップ5**: | 死因 | 発生確率(高/中/低) | 致命度 | 今日打てる対策 or 早期発見の兆候 |
- **Step3. 因果チェーン**: 上位2死因について「最初の小さな判断→増幅→死」の流れをテキスト図で
- **Step4. 監視ダッシュボード**: 失敗の予兆として毎週見るべき指標・状態を3〜5個(測定方法付き)
- **Step5. 撤退ライン**: 「これが起きたら計画を止めて再検討する」条件を1つ以上、測定可能な形で
- **Step6. それでも進む理由**: 対策済みリスクを踏まえた、進む価値の再確認1文(進むべきでないなら正直にそう書く)

最後に必ず添える:
> **死因リストは呪いではなく保険です。対策した上で、堂々と進んでください。最終判断は現場を知る人間のものです。**

## 品質基準(すべて満たすまで完成としない)

- [ ] 失敗の定義が数字・状態で具体的に書かれ、性質の異なる2シナリオがある
- [ ] 死因が8つ以上列挙され、上位死因は構造レベルまで掘られている
- [ ] 「そもそも始めるべきだったか」を通過している
- [ ] 推進者役が自己開示の証言をしている(外部要因への責任転嫁で終わっていない)
- [ ] 対策が「予防」と「早期発見」に分類されている
- [ ] 撤退ラインが測定可能な形で定義されている
- [ ] 5人目のキャラが実際に発言している
