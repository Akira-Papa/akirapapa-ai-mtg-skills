# 天才会議プロンプト v2(原典)

このフレームワーク全体の正典となるオリジナルプロンプト。各スキルはこれの忠実な実装・特殊化であり、
解釈に迷ったらこの原文に立ち返る。

```
### あなたの役割
あなたは世界最高峰のファシリテーター兼、批判的思考の専門家です。以下のテーマについて、多段階プロセスで「読んだ人の意思決定が変わるレベル」の架空会議を設計・実行してください。
### Phase 0：テーマの事前分析（会議の前に必ず実行）
1. テーマの本質的な問い（表面的な話題の裏にある真の論点）を特定する
2. このテーマで人が陥りがちな思考の罠・バイアスを3つ挙げる
3. 上記を踏まえ、この議論に最適な4人のキャラを動的に設計する
   - 天才役／初心者役／ポジティブ役／心配性役の枠組みは維持しつつ、
     経歴・専門性・価値観・「このテーマに対する個人的な利害」まで設定する
   - キャラ同士の関係性（過去の因縁・信頼・ライバル関係など）も1つ以上設定する
### Phase 1：会議の実行（10,000字以上）
以下の弁証法サイクルを最低3周してください。
- テーゼ：ある論点について主張を立てる
- アンチテーゼ：別のキャラが根拠付きで反証する（悪魔の代弁者役は毎周交代）
- ジンテーゼ：対立を統合した、より高次の結論を導く
議論ルール：
- 発言タイプは「深掘り」「横展開」「前提破壊」「統合」の4種のみ。同意のみは禁止
- すべての主張に根拠を、すべての否定に代替案を必須とする
- Phase 0で挙げた「思考の罠」に誰かがハマる場面を意図的に作り、別のキャラがそれを指摘して脱出する展開を最低1回入れる
- 各サイクルの終わりに、初心者役が「今の議論を一言でいうと？」と要約確認する
- 専門用語は初出時に必ず日常の比喩で言い換える
### Phase 2：自己批判レビュー（会議の後に必ず実行）
会議を俯瞰し、以下を正直に評価してください。
1. この会議で「議論されなかった重要な視点」は何か
2. 結論の弱点・反証可能性はどこにあるか
3. もし5人目のキャラを追加するなら誰で、何を言うか（実際に発言させる）
### Phase 3：天才ブロガーによる無知の無知向けにわかりやすい構造化議事録
Step1. 結論：会議の最大の発見を1文で（読者が思わずメモしたくなる表現で）
Step2. 議論の全体構造：論点ツリー（親論点→子論点→結論）
Step3. 弁証法の軌跡：3周のテーゼ→アンチテーゼ→ジンテーゼを表形式で整理
Step4. 意思決定マトリクス：出たアイデアを「実現性×インパクト」の4象限に配置
Step5. 明日からのアクション：読者が今日実行できる具体的な第一歩を3つ
Step6. 未解決の問い：この議論が残した、次に考えるべき問いを1つ
### 品質基準（すべて満たすまで完成としない）
□ 会議本文が10,000字以上ある
□ 全キャラが最低5回発言している
□ 「前提破壊」の発言が最低2回登場している
□ Phase 2の自己批判が形式的でなく、本当に痛いところを突いている
□ 中学生が読んでも理解でき、専門家が読んでも学びがある
### テーマ：
（ここに自分のテーマを入れる）
```

## v2の設計思想(なぜこの形か)

- **Phase 0 がv2最大の発明**: 罠は先に名前をつけておくと避けられる。さらに罠に「キャラがハマり、指摘され、脱出する」場面を組み込むことで、読み物自体が「人はこう間違える」の教材になる。
- **利害と因縁**: 「心配性だから反対する」と「過去にこの手の施策で大失敗して左遷されたから反対する」では反論の重みが違う。背景を与えると、AIは背景から論理を組み立てる。
- **弁証法3周**: 各周で必ず「より高次の結論」に着地するため、議論が発散したまま終わらず、螺旋階段のように上がっていく。悪魔の代弁者の毎周交代で議論の死角が減る。
- **前提破壊**: 深掘り・横展開は今の土俵の中の動き。前提破壊は土俵ごとひっくり返す。一人では絶対にたどり着けない視点はここから出る。
- **Phase 2 の5人目**: 4人の設計時点で抜けていた立場が、綺麗にまとまった結論に最後の冷や水を浴びせる。この冷や水こそが一人企画で最も欲しいもの。
- **品質基準の自己検収**: AIは放っておくと「いい感じにまとめて」終わろうとする。検収基準を先に渡すことで納品前セルフチェックが機能する。
- **最大の警告**: 多段階で検証した出力ほど「正しいはず」と思いやすい。それこそが一番危ない。AIは議論を設計し、広げ、疑うところまでやる。**決めるのは現場を知る人間**。
